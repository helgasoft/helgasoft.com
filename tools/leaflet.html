If your kml file or gpx file or geojson file is too big, you can compress (reduce) the file size. It is called optimize the file. Lets explain how to prune a kml/gpx/geojson file and make it smaller, condensed, truncated.

Truncate Geo Tracks Live!

Smaller geospatial files are delivered faster on the Internet. Some GPS devices and mapping software generate large files with excessive number of points. That much accuracy is usually redundant and our application is reducing size by trucating the number of track points. Size reduction of over 90% is possible.

The application was built with R using Shiny and the Leaflet library for mapping. Truncation (pruning) was implemented by the Douglas-Peucker algorithm. In-depth XML manipulations were performed thanks to the xml2 library.

File types supported:

  • gpx - generated by GPS-enabled devices
  • kml - open file format by Google
  • geojson - geographical elements with additional attributes in JSON

Notes and Limitations

Tracks with 1000s of points will take time. An animated icon () appears during processing. Elevation data is supported along with latitude and longitude.
Additional elements from the file, like waypoints or shapes, may be shown along with the tracks. They are copied unchanged in the output file.

  • kml
    Elements Polygon, LineRing, MultiGeometry are not supported. Duplicate coordinate points will be deleted.

  • gpx
    Track points time (<trkpt> <time>) is not saved in the output GPX file. Leaflet shows route points (<rtept>) connected by lines, which could make the map appear congested.
